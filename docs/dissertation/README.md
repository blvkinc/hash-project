## Required Shape

The project brief expects a standalone dissertation of approximately 15,000 words.
The marking scheme gives most weight to product quality, literature review,
methodology, results/discussion, and the ability to demonstrate project ownership.

The baseline document therefore follows this chapter model:

1. Introduction
2. Literature Review and Background
3. Methodology
4. Design and Implementation
5. Evaluation and Results
6. Discussion
7. Conclusion and Future Work

## Writing Position

The dissertation should not present the project as only a hash monitor. The
central argument is that developer machines and small organisations need a
local-first file integrity system that can detect meaningful compromise signals,
especially reverse-shell/RCE payloads and compromised software dependencies,
without creating alert fatigue.
